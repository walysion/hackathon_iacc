Propósito del SistemaTalita Kum es una plataforma web móvil-first diseñada para optimizar la recolección, estructuración y análisis de datos clínicos en terreno mediante intervención asistida por Inteligencia Artificial. La aplicación permite a los terapeutas documentar casos complejos mediante la voz, mientras que proporciona a la jefatura un panel de Business Intelligence (BI) interactivo para la toma de decisiones estratégicas.2. Arquitectura del Sistema y Flujo de DatosEl sistema se rige por un orquestador centralizado de estados en el componente raíz (App.vue). No utiliza routers pesados para maximizar la velocidad en dispositivos móviles de baja gama en terreno; en su lugar, utiliza un control de pasos reactivo (currentStep).Flujo de Usuario y Roles IntegradosAutenticación: El usuario ingresa credenciales reales verificadas por Firebase Auth.Segundo Factor (2FA): Se exige un token simulado de 6 dígitos con teclado numérico nativo forzado para dispositivos móviles.Filtro de Roles: El sistema analiza si el correo contiene el término "admin":Super Admin / Jefatura: Accede a una pantalla intermedia de Selección de Módulo (role-select) que le permite alternar libremente entre el Panel de Supervisión y el Panel de Terreno.Terapeuta Estándar: Salta la selección y va directo al Panel de Terreno.4. Estructura Completa de Archivos Maestros📁 src/firebase.js (Conexión Base)Establece la conexión directa con el proyecto real en la nube de Google (talitakum-hackathon). Exporta la instancia auth utilizada por los formularios de acceso.📁 src/App.vue (Cerebro y Orquestador)Controla la navegación mediante los pasos reactivos del MVP (login ➔ role-select ➔ admin-dashboard / dashboard ➔ capture ➔ review ➔ success). Mantiene el Estado Global de Actividades (globalActivities), permitiendo que las nuevas intervenciones grabadas por voz se inyecten dinámicamente en el historial del terapeuta en tiempo real.📁 src/components/LoginView.vue (Acceso Seguro)Formulario con validación nativa. Captura excepciones reales de Firebase (credenciales inválidas, bloqueos por reintentos). Cuenta con filtrado numérico estricto en tiempo real para el token 2FA y un botón de retorno seguro para corregir correos mal digitados.📁 src/components/AdminDashboard.vue (Panel de Jefatura / BI)Muestra KPIs corporativos que reaccionan dinámicamente cuando ocurren altas o registros. Integra un gráfico de dona estilizado para Glassmorphism con tipografías legibles. Cuenta con un Módulo CRUD de Terapeutas con Innovación de MVP: genera contraseñas aleatorias seguras en texto plano y levanta de forma nativa un disparador mailto: con la plantilla del correo de bienvenida redactada automáticamente para el nuevo funcionario.📁 src/components/DashboardView.vue (Panel del Terapeuta)Diseñado para su uso ágil en terreno. Consume la propiedad dinámica de actividades enviada por el orquestador. Presenta tarjetas resumidas de rendimiento y un botón central de acción masiva para iniciar la captura de voz de manera inmediata.📁 src/components/AudioCapture.vue (Captura de Voz con IA)Utiliza la API nativa de reconocimiento de voz del navegador (SpeechRecognition). Cuenta con modificaciones senior críticas:continuous = true: Permite pausas naturales en el relato del terapeuta sin cortar la grabación.onUnmounted: Limpieza activa de hardware para evitar que el micrófono se quede encendido si el usuario abandona la vista.Flujo de precarga con animación (Loader) para emular la latencia de procesamiento de un modelo de lenguaje (LLM).📁 src/components/InterventionReview.vue (Validación Clínica)Muestra los campos obligatorios estructurados por la IA en un formato editable por el profesional (Objetivo, Desarrollo, Acuerdos, Acciones, Observaciones). Cuenta con protección de scroll para evitar deformaciones en la tarjeta contenedora y una utilidad de Bono de Innovación que genera una cadena formateada para compartir reportes de incidentes inmediatamente vía WhatsApp Web o Móvil.5. Resumen de Hitos Logrados (En Qué Quedamos)Conexión real establecida: La autenticación se comunica directamente con los servidores activos de Firebase.Flujos reactivos conectados: El micrófono inyecta texto real en el formulario de edición, y el formulario de edición actualiza el historial global del Dashboard de manera exitosa.Seguridad de administrador resuelta: La jefatura ya no queda atrapada en los gráficos; cuenta con un menú selector de módulos y un botón global en el encabezado para alternar entre la supervisión técnica y el registro clínico.Producción Activa: La aplicación fue empaquetada con éxito mediante npm run build y se encuentra desplegada en los servidores de Netlify de forma pública.6.
